<h2 id="How%20it%20works%3F">How it works?</h2>
Every HTML page has:
<ul><li><code>&#60;meta http&#45;equiv&#61;&#34;refresh&#34; content&#61;&#34;0;url&#61;&#46;&#46;&#46;&#34;&#62;</code> in <code>&#60;head&#62;</code>. This means that the user will be redirected to the destination URL in 0 seconds after the page has loaded.</li><li>Fallback #1 - <code>&#60;script&#62;&#46;&#46;&#46;&#60;/script&#62;</code> in <code>&#60;body&#62;</code> redirects the user to the destination URL.</li><li>Fallback #2 - Other elements in <code>&#60;body&#62;</code> ("Redirecting... <...>", "Didn't get redirected? <code>&#60;a &#46;&#46;&#46;&#62;</code> Click here! <code>&#60;/a&#62;</code> ").</li></ul>
That means that users should be redirected, even if they have disabled JavaScript in their browser settings.

<h2 id="Why%20is%20%3Ccode%3EHTTP%203XX%3C%2Fcode%3E%20better%3F">Why is <code>HTTP 3XX</code> better?</h2>
A response with an <code>HTTP 3XX</code> status code and with the <code>location</code> header makes a real redirect.
