<p><h1 id="Postprocessor%20mode">Postprocessor mode</h1>
<strong>- Add your own files to generated Next.js website.</strong>

<blockquote>With this mode you can add your own files to generated Next.js website. <br>This mode creates the <code>deploy</code> directory and outputs files into it.<br> If your Next.js website outputs an <code>en&#46;html</code> file, it will be copied into the <code>index&#46;html</code> file.</blockquote>
This mode requires the <code>just&#46;config&#46;js</code> file, the workflow file, some directories and <code>&&#35;95;just/404&#46;html</code>.
Next.js website should be generated <strong>before</strong> running this mode.
Generated Next.js website should be in <code>&#46;next/server/pages/</code> and <code>&#46;next/static/</code> directories.
Required directories are:
<ul><li><code>&&#35;95;just</code></li><li><code>&&#35;95;just/dangerously&#45;insert&#45;files</code></li><li><code>&&#35;95;just/js</code></li><li><code>&&#35;95;just/style</code></li></ul>

<span class="cubRKcR"><code>just&#46;config&#46;js</code></span>

<span class="cubRKcR"><code>&#46;github/workflows/WORKFLOW&&#35;95;NAME&#46;yml</code></span>

<h2 id="Required%20directories">Required directories</h2>
<span class="cubRKcR">Required only directories, not files within them (except for the <code>&&#35;95;just/404&#46;html</code> file).</span>
In <strong><code>&&#35;95;just/js</code></strong> you can put your JavaScript files to be inserted into the <code>deploy/&&#35;95;just</code> directory and into every HTML page.
In <strong><code>&&#35;95;just/style</code></strong> you can place your CSS files to be inserted into the <code>deploy/&&#35;95;just</code> directory and into every HTML page as well.
Within <strong><code>&&#35;95;just/dangerously&#45;insert&#45;files</code></strong>, you may include any files that you wish to insert into the <code>deploy</code> directory.

<blockquote class="mTaeV">Inserting files via the <strong><code>&&#35;95;just/dangerously&#45;insert&#45;files</code></strong> directory may cause website errors or files may not be inserted if there is already a file with the same name in the <code>deploy</code> directory.</blockquote>
The <strong><code>&&#35;95;just/404&#46;html</code> file</strong> is required and it'll become the <code>deploy/404&#46;html</code> file (if the <code>deploy/404&#46;html</code> file already exists, it will be overwritten).

<h2 id="How%20it%20works%3F2">How it works?</h2>
<ol><li>It creates a <code>deploy</code> directory and copies every file from the <code>&#46;next/server/pages</code> and <code>&#46;next/static</code> directories to it.</li><li>If there is a <code>deploy/en&#46;html</code> file, it will copies to the <code>deploy/index&#46;html</code>.</li><li>It copies <code>&&#35;95;just/404&#46;html</code> to <code>deploy/404&#46;html</code> (if it exists, it'll be overwritten).</li><li>Every <code>&&#35;95;just/js/&&#35;42;&#46;js</code> copies into <code>deploy/&&#35;95;just</code>; Every <code>&&#35;95;just/style/&&#35;42;&#46;css</code> copies into <code>deploy/&&#35;95;just</code>; Every <code>&&#35;95;just/dangerously&#45;insert&#45;files/&&#35;42;</code> copies into <code>deploy</code>.</li><li>Every JavaScript and CSS file in <code>deploy/&&#35;95;just</code> are inserted into every HTML page as HTML tags.</li></ol>

<h2 id="Troubleshooting">Troubleshooting</h2>
This mode may cause website issues.
You can try fixing them by switching the postprocessor version in your workflow file.
<_just element="cubRsK">16</_just>
Available postprocessor versions are: <code>&#34;24&#34;</code>, <code>&#34;26&#34;</code> (default), <code>&#34;32&#34;</code>.
