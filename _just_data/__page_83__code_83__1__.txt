This documentation assumes some familiarity with
- GitHub
- GitHub Actions
- GitHub Pages
And some familiarity with these languages
- JavaScript
- YAML
- HTML
- CSS
- Markdown
<span class="language-xml"><span class="hljs-tag">&lt;<span class="hljs-name">h2</span>&gt;</span></span>Installation<span class="language-xml"><span class="hljs-tag">&lt;/<span class="hljs-name">h2</span>&gt;</span></span>
<span class="language-xml"><span class="hljs-tag">&lt;<span class="hljs-name">h3</span>&gt;</span></span>Making your first project<span class="language-xml"><span class="hljs-tag">&lt;/<span class="hljs-name">h3</span>&gt;</span></span>
- Create new repository, and create `/.github/workflows/publish.yml` file, template:
&amp;#96;&amp;#96;&amp;#96;yml
name: Website

on:
  push:

 branches: [&quot;main&quot;]
  workflow_</span>dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: &quot;pages&quot;
  cancel-in-progress: false

jobs:
  build:

 runs-on: ubuntu-latest

 steps:

<span class="hljs-bullet">  -</span> name: Checkout

<span class="hljs-code">    uses: actions/checkout@v4
</span>
<span class="hljs-bullet">  -</span> name: Setup Pages

<span class="hljs-code">    uses: actions/configure-pages@v5
</span>
<span class="hljs-bullet">  -</span> name: Generate with <span class="hljs-emphasis">_just

    uses: js-just/latest@main

    with: # Remove &quot;with&quot; and &quot;path&quot; here if you are not using compressor or generator modes!

      path: . # Root directory, or you can replace the dot with the path to your website/docs directory to be generated/compressed. (Only for compressor and generator modes)

  - name: Upload artifact

    uses: actions/upload-pages-artifact@v3

    with:

      path: . # Root directory, or you can replace the dot with the path to your entire website to be deployed to GitHub Pages.

  deploy:

 environment:

  name: github-pages

  url: ${{ steps.deployment.outputs.page_</span>url }}

 runs-on: ubuntu-latest

 needs: build

 steps:

<span class="hljs-bullet">  -</span> name: Deploy to GitHub Pages

<span class="hljs-code">    id: deployment
</span>
<span class="hljs-code">    uses: actions/deploy-pages@v4
&amp;#96;&amp;#96;&amp;#96;
You can change name of workflow file and workflow name:
&amp;#96;&amp;#96;&amp;#96;yml
name: Workflow name
&amp;#96;&amp;#96;&amp;#96;
You can also choose _just version:
&amp;#96;&amp;#96;&amp;#96;yml
</span>
<span class="hljs-code">    uses: js-just/_just@(put version name here)
&amp;#96;&amp;#96;&amp;#96;
**If you know what exactly you are doing, you may change anything.**
- Create `just.config.js` file in the root directory:
Choose what mode you want to use.
 
Using `Postprocessor` mode:
&amp;#96;&amp;#96;&amp;#96;js
module.exports = {
  type: &quot;postprocessor&quot;
}
&amp;#96;&amp;#96;&amp;#96;
Using `Redirector` mode: 
&amp;#96;&amp;#96;&amp;#96;js
module.exports = {
</span>
 type: &quot;redirect&quot;, 

 redirect<span class="hljs-emphasis">_config: {

    url: &quot;https://example.com/&quot;, // Required. Replace with destination URL.

 }
}
&amp;#96;&amp;#96;&amp;#96;
Using `Compressor` mode:
&amp;#96;&amp;#96;&amp;#96;js
module.exports = {

 type: &quot;compress&quot;
}
&amp;#96;&amp;#96;&amp;#96;
Using `Generator` mode:
&amp;#96;&amp;#96;&amp;#96;js
module.exports = {

 type: &quot;docs&quot;,

 docs_</span>config: {

<span class="hljs-code">    metatitle: &quot;Documentation title&quot;, // Required. Replace with your documentation title.
</span>
<span class="hljs-code">    domain: &quot;example.com&quot; // Required. Replace with your domain name. Domain name should be valid.
</span>
 }
}

<span class="hljs-section">- Read the documentation for the mode that you&#x27;ve chosen.

<span class="language-xml"><span class="hljs-tag">&lt;<span class="hljs-name">h3</span>&gt;</span></span>Pro installation<span class="language-xml"><span class="hljs-tag">&lt;/<span class="hljs-name">h3</span>&gt;</span></span>
<span class="hljs-bullet">-</span> Create or modify your <span class="hljs-code">`.github/workflows/github_pages_workflow_name.yml`</span>:
Make sure that permissions allow writing <span class="hljs-code">`pages`</span> and <span class="hljs-code">`id-token`</span>, but do not allow writing <span class="hljs-code">`contents`</span>, only read.
<span class="hljs-literal">&amp;#96;</span><span class="hljs-literal">&amp;#96;</span><span class="hljs-literal">&amp;#96;</span>yml
permissions:
  contents: read
  pages: write
  id-token: write
<span class="hljs-literal">&amp;#96;</span><span class="hljs-literal">&amp;#96;</span><span class="hljs-literal">&amp;#96;</span>
Make a job for building your website using <span class="hljs-emphasis">_just:
&amp;#96;&amp;#96;&amp;#96;yml
jobs:
  build:

 runs-on: ubuntu-latest

 steps:

  - name: Checkout

    uses: actions/checkout@v4

  - name: Generate with _</span>just

<span class="hljs-code">    uses: js-just/_just@ # version name (recommended) (example: v0.0.29) / main branch (latest commit) (unstable, not recommended) / commit SHA (not recommended)
</span>
<span class="hljs-code">    with:
</span>
<span class="hljs-code">      path: # Path to your website directory to be generated/compressed. (Only for compressor and generator modes)
&amp;#96;&amp;#96;&amp;#96;
- Create `just.config.js` file:
Basic usage:
&amp;#96;&amp;#96;&amp;#96;js
module.exports = {
  type: &quot;(postprocessor/redirect/compress/docs)&quot;
}
&amp;#96;&amp;#96;&amp;#96;
Using multiple modes:
&lt;span class=&quot;loueCmc&quot;&gt;Currently available only combining generator and compressor.&lt;/span&gt;
&amp;#96;&amp;#96;&amp;#96;js
module.exports = {
  type: [&quot;docs&quot;, &quot;compress&quot;]
}
&amp;#96;&amp;#96;&amp;#96;
- Read the documentation for the mode/modes that you&#x27;ve chosen.
